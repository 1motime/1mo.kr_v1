Project Context

비즈넵이 V3.0으로 버전업 하며 달라진점들 중 하나는 사용자 메트릭을 수집할 수 있게 개발 구조가 개선된 점이었다. 이전 버전까지는 빠른 MVP 구현과 굵직한 KPI 달성에 초점을 두었다. 따라서 결과물 제작 위주로 업무가 진행됐고 사용자 데이터를 수집·분석하여 개발에 반영하기 어려웠다. V3.0으로 접어들며 핵심 기능이 하나 둘 개발 완료되니 기능 안정화가 필요하다는 목소리가 대두됐다. 나는 문제 정의할 때 사용자 데이터를 활용하고 싶었고 문제 해결 후에도 결과를 분석 할 수 있는 지표가 필요했다. 사실 데이터 기반 결정 구조나 그로스 해킹 등은 IT 업계에서 이미 상식처럼 자리잡은 상황이었다. 우리도 여러가지 상황탓을 그만하고 그 표준에 하루 빨리 다가서야하는 실정이었다. 마침 사장님을 대상으로 디스플레이 광고를 진행하는 업무를 맡았다. 광고로인한 KPI 향상 결과 측정을 위해 시작된 해당 프로젝트로 UTM 추적은 물론 전환율 파악, 퍼널 분석, 사용자 이벤트 분류 및 관리가 가능한 구조가 만들어졌다. 덕분에 사용자 데이터를 UX디자인에 반영 할 수 있는 초석을 다진 것 같아 뿌듯한 프로젝트였다.
